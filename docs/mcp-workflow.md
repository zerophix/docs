# MCP Filesystem 配置流程说明

## 什么是 MCP？

MCP（Model Context Protocol，模型上下文协议）是一个标准协议，让 AI 助手可以通过统一的接口调用外部工具。就像 USB 标准让电脑可以连接各种外设一样，MCP 让 AI 可以连接各种工具和服务。

## 分步详解

### 第①步：读取配置
我通过读取 `cline_mcp_settings.json` 文件，发现你配置了 `filesystem` 这个 MCP 服务器。配置中指定了：
- 启动命令：`npx`
- 命令参数：`-y @modelcontextprotocol/server-filesystem /Users/mingxilv/dolphinmind`
- 状态：启用（`disabled: false`）

### 第②步：启动子进程
Cline 扩展根据配置，在你的电脑上启动了一个**独立的 Node.js 进程**，运行官方提供的 `@modelcontextprotocol/server-filesystem` 包。这个进程专门负责文件操作。

### 第③步：JSON-RPC 通信
主进程（Cline）和子进程（filesystem 服务器）通过 **stdin（标准输入）和 stdout（标准输出）** 管道进行通信，使用 JSON-RPC 2.0 协议：
- **Cline → 服务器**：通过 stdin 发送 JSON 格式的请求（列出工具 / 调用工具）
- **服务器 → Cline**：通过 stdout 返回 JSON 格式的响应（工具列表 / 执行结果）

### 第④步：文件操作
当 filesystem 服务器收到调用请求（比如 `read_text_file`）后，它使用 Node.js 的 `fs` 模块直接操作你的本地文件系统，读取或写入文件。

## 关键设计理念

| 特点 | 说明 |
|------|------|
| **安全隔离** | filesystem 服务器是独立进程，只允许访问你在参数中指定的目录 |
| **标准化** | 所有 MCP 服务器都遵循同一协议，我可以用同样方式调用天气、数据库、GitHub 等各种工具 |
| **热插拔** | 修改配置文件后无需重启 VS Code，扩展会自动重新加载 MCP 服务器 |
| **声明式配置** | 你只需在 JSON 中声明要启动什么服务器，不需要写任何代码 |

## 一句话总结

> **我通过 MCP 协议，让一个专门的文件管理程序（filesystem 服务器）帮我操作你的文件，而不是我直接读写文件系统。** 这就像你让一个专业的图书管理员帮你找书，而不是自己跑进书库乱翻——更安全、更规范、更可靠。
