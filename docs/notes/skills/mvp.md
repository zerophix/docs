# Anti-Impatience MVP Protocol (v2.0)

## 🎯 核心目标

通过**强制微小化**（Micro-steps）和**可视化进度**，帮助开发者克服“急躁”心理，实现平滑、高质量的持续交付。

---

## ⚙️ 强制执行流程 (The 5-Step Loop)

When user inputs `/mvp [feature_name]`, AI must guide through:

### Step 1: 定义“丑陋”初版 (Define Ugly First)
**提问**：这个功能最核心的“一个动作”是什么？
**要求**：列出 `Must Have`（必须有）和 `Out of Scope`（暂不做）。
**原则**：如果初版看起来不够简单，说明还不够 MVP。

### Step 2: 拆解微任务 (Micro-Tasks)
**规则**：每个任务执行时间 ≤ 30 分钟。
**输出**：生成一个包含 3-5 个微任务的清单，每个任务都有明确的产出物（如：“Order 实体类编译通过”）。

### Step 3: 计时冲刺与提交 (Sprint & Commit)
**指令**：启动 25 分钟倒计时。
**约束**：每完成一个微任务，必须立即执行 `git commit`。
**格式**：`[MVP] Task N/Total: [任务简述]`

### Step 4: 快乐路径测试 (Happy Path Test)
**重点**：只验证核心流程是否跑通。
**跳过**：性能测试、边界条件、安全加固全部留到 v2。

### Step 5: 债务记录与迭代 (Debt & Iterate)
**动作**：将本次为了速度而妥协的地方记入“技术债务清单”，并规划 v2 优化点。

## 📝 交互模板

```markdown
## 🚀 MVP 启动: [功能名称]

### 1. 核心定义
- **唯一目标**: ...
- **暂不处理**: ❌ ...

### 2. 微任务清单 (Est. < 2h)
- [ ] Task 1: ... (Output: ...)
- [ ] Task 2: ... (Output: ...)

### 3. 执行状态
⏱️ **当前冲刺**: Task 1/3
📊 **进度条**: [██░░░] 20%

### 4. 债务记录
- ⚠️ [待优化点 1]
```

## 🛠️ 路径与规范

| 资源 | 说明 |
| :--- | :--- |
| **提交规范** | 必须以 `[MVP]` 为前缀，方便后续统计迭代频率 |
| **停车区** | 开发中产生的新想法必须存入 `PARKING-LOT.md`，严禁中途插入 |
| **庆祝机制** | 完成 MVP 后，AI 应给予正向反馈并建议短暂休息 |

---

## 💡 应对急躁策略

1.  **两分钟法则**：想加新功能时，先写进停车区，继续当前任务。
2.  **物理锚点**：感到焦虑时，起立深呼吸，重申“One task at a time”。
3.  **视觉激励**：每勾选一个任务，都是一次多巴胺奖励。
